tool 을 사용하면 Claude 가 외부 정보에 액세스 할 수 있다.
Claude 는 기본적인 학습 데이터에서만 정보를 알고 있다. 실시간성이 높은 데이터와 외부 시스템에 접근할 수 없음. 때문에 tool 을 통해서 이를 해소 시켜 줄 수 있다.

당연히, 최신 정보뿐만 아니라 DB, API 등 기타 서비스에 연결해서 더 높은 품질의 답변이 가능해진다.
예를 들어, "캘리포니아 지금 날씨가 어때?" 라고 물으면, Claude 는 대답이 불가능하다.

## How Tool Use Works
<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/0b554262-a578-4e9e-872d-cc0fa38944a3" />

1. 초기 요청
   Claude 에게 외부 출처에서 추가 데이터를 얻는 방법에 대한 질문과 함께 안내를 보낸다.
2. tool 요청
   Claude 는 질문을 분석한 후 추가 정보가 필요하다고 판단한 뒤, 어떤 데이터가 필요한지 구체적으로 요청한다.
3. 데이터 검색
   서버는 외부 API 나 데이터베이스에서 요청된 정보를 가져오는 코드를 실행한다.
4. 최종 답변
   검색된 데이터를 Claude 에 다시 보내면, Claude 는 원래 질문과 새로운 데이터를 모두 사용하여 완전한 답변을 생성한다.

결국 Claude 에게 추상적인 질문을 던지면, Claude 가 어떤 데이터가 필요한지 구체화 하고, 중계 서버가 직접 그 데이터를 구해서 던져주면, Claude 가 참고해서 해결해주는 방식.

## 예시 상정 - 일정 리마인더

"병원 진료 예약을 알려주세요. 일주일 후 16:00 이에요."
이 기능을 사용하려면 Claude 가 시간과 알림을 관리하는데 있어 몇 가지 한계가 있음을 인지해야 한다.

- 제한된 시간 인식
  Claude 는 현재 날짜는 알지만 정확한 시간은 모를 수 있다.
- 날짜 계산 문제
  Claude 는 특히 여러 날 후를 생각할 때 시간 기반 덧셈을 항상 잘 처리하지 못한다.
- 알림 기능 없음
  Claude 는 알림 설정법을 모른다. (내장 메커니즘이 없음)

### tool 함수 제공하기
Python 을 이용해 Claude 와 소통하는 환경에서, Python function 을 생성해서 Claude 에게 도움을 줄 수 있다.
가령 오늘 날씨를 제공하는 function 을 아래와 같이 작성하면, Claude 가 이를 활용할 수 있다.

```python
def get_weather(location):
	if not location or location.strip() == "":
		raise ValueError("Location cannot be empty")
		
	url = "https://fakeweatherapi.example.com/current"
	params = {
		"q": location,
		"appid": api_key,
		"units": "metric"
	}
	
	response = requests.get(url, params, timeout=10)
	response.raise_for_status()
	
	return response.json()
```

function 을 완성하는 방법은 각 언어, 원하는 기능에 따라 다르다. 집중해서 봐야하는 파트는 예외처리 부분.
**예외를 명확하고 구조적으로 처리해 줄 수록 Claude 가 호출 중 에러를 마주했을 때 어떻게 다시 호출해야하는지 빠르게 학습하고 응용할 수 있다.**

함수를 그냥 덩그러니 제공하면, Claude 가 그런 함수가 존재한다는 걸 인지하기 어려울 것이다. 이에 대한 도움을 주기 위해 tool schema 를 JSON 형태로 말아서 제공해준다.

```json
{
	"name": "get_weather",
	"description": "Retrieves current weather...",
	"input_schema": {
		"type": "object",
		"properties": {
			"location": {
				"type": "string",
				"description": "The location for which..."
			}
		},
		"required": ["location"]
	}
}
```

JSON 을 어렵게 생각할 필요가 없다. Claude 가 function 을 호출하는데 필요한 정책, 유효성 검사를 미리 인지할 수 있도록 function 의 존재와 이름, 그리고 사용 규칙을 미리 안내하는 것이다.

그럼 tool function 을 준비할 때마다 JSON schema 도 한 벌씩 꼭 준비해야하는가? 일일이 손으로?
다행히 이런 건 Claude 한테 요청하면 된다.

```
아래 함수에 대해 JSON schema spec 을 준비해주세요.
Anthropic Tool use with Claude 문서 내용을 참고해서 작성을 진행해주세요.
```

작성된 JSON schema 에 대해 아래와 같이 anthropic.types 로 감싸주면 타입 안정성이 더 올라간다.

```python
from anthropic.types import ToolParam

get_current_datetime_schema = ToolParam({ 
	"name": "get_current_datetime", 
	"description": "Returns the current date and time formatted according to the specified format", # ... rest of schema
})
```

## Running the Tool Function
당연히 함수가 준비되었다면, Claude 가 이를 어떻게 실행 시키는지 또한 알아야 한다.
응답에서 tool function 파라미터에 접근하는 방법은 다음과 같다.

```python
response.content[1].input
```

 python unpacking 문법을 이용하면 아래와 같이 사용할 수 있다.

```python
get_current_datetime(**response.content[1].input)
```

tool function 을 실행한 이후, results block 을 사용해서 결과를 claude 에게 돌려보내야 한다.

```python
{
	"tool_use_id": "toolu_01BEbi7q7qz",
	"type": "tool_result",
	"content": "12:47:13",
	"is_error": False
}
```

`tool_use_id` 로 식별하는 덕분에 여러 tool 호출을 동시에 진행해도 각각 구별이 가능하다.

그리고, 역시나 Claude 는 대화이력을 기억하지 않으므로 대화를 다음과 같이 기록해두어야 한다.

```python
messages.append({
	"role": "user", 
	"content": [{ 
		"type": "tool_result", 
		"tool_use_id": response.content[1].id, 
		"content": "15:04:22", 
		"is_error": False
	}]
})
```

이후에도 JSON schema 는 대화에 포함시켜야한다. Claude 가 다른 tool 을 호출하지 않을 거라고 예상하더라도, Claude 자체가 대화 내역 중 tool 참조를 이해하기 위해서 필요하다.

```python
client.messages.create( 
	model=model, 
	max_tokens=1000, 
	messages=messages, 
	tools=[get_current_datetime_schema]
)
```

## 여러 개의 Tool 연결하기(Multi-Turn Tool Pattern)
결국 지향점은 여러 개의 Tool 을 동시에 연결하는 것이다. 무슨 일이 벌어질까?

"오늘로부터 103일 후는 무슨 날인가요?" 라는 질문을 던진다면, Claude 는 먼저 현재 날짜를 받아내고 그 다음 103일을 더해주어야 한다. 그로 인해 Claude 가 오늘 날짜를 얻는 Tool 과 103일을 더하는 Tool 을 두 차례 반복 호출해서 작업을 수행한다. 그림으로 표현하면 아래와 같아짐.

<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/717d7841-97ae-42d7-8194-9c64c11f2ee4" />

이를 우리는 **Multi-Turn Tool Pattern** 이라고 부른다.
당연히, 이를 자연스럽게 지원하기 위해선 서버 쪽 코드에 루프문이 필요하다.

```python
def run_conversation(messages):
	while True:
		response = chat(messages)
		add_assistant_message(messages, response)
		
		# Pseudo code
		if response isn't asking for a tool:
			break
			
		tool_result_blocks = run_tools(response)
		add_user_message(messages, tool_result_blocks)
		
	return messages
```

본격적으로 Multi-Turn Tool Pattern 을 지원하기 위해선 함수 리팩터링이 필요하다.

```python
from anthropic.types import Message 

def add_user_message(messages, message): 
	user_message = { 
		"role": "user", 
		"content": message.content if isinstance(message, Message) else message
	} 
	messages.append(user_message)
```

```python
def chat(messages, system=None, temperature=1.0, stop_sequences=[], tools=None):
	params = { 
		"model": model, 
		"max_tokens": 1000, 
		"messages": messages,
		"temperature": temperature, 
		"stop_sequences": stop_sequences,
	 } 
	 
	 if tools: 
		 params["tools"] = tools
	 if system: params["system"] = system
	 
	 message = client.messages.create(**params)
	 return message
```

```python
def text_from_message(message):
	return "\n".join(
		[block.text for block in message.content if block.type == "text"]
	)
```

루프 코드는 준비됐는데, Claude 가 Tool 을 요구하는지 아닌지는 어떻게 감별할까?
Claude 의 response 에 `stop_reason` 필드를 참조하면 된다.

```python
if response.stop_reason != "tool_use":
	break # Claude is done, no more tools needed
```

고로, 루프문의 수도 코드를 완성하면 다음과 같아진다.

```python
def run_conversation(messages):
	while True:
		response = chat(messages, tools=[get_current_datetime_schema]) 
		add_assistant_message(messages, response) 
		print(text_from_message(response)) 
		
		if response.stop_reason != "tool_use": 
			break 
		
		tool_results = run_tools(response) 
		add_user_message(messages, tool_results) 
	
	return messages
```

Claude 의 Tool 요청은 한 번에 하나씩이 아니다. 여러 Tool 을 한 번에 요청할 수 있다. response 에는 `ToolUseBlock` 이 포함되어 있어서, 각각의 알맞는 함수로 처리를 해주면 된다.

당연하게도, try-catch 문으로 오류 관련 안정성 처리를 해주면 더 좋다.

## Claude 가 기본적으로 제공하는 Tool 들

Claude 가 기본적으로 제공해주는 Tool 들도 존재한다. **정확히는 Tool 구현체가 아닌 JSON schema 를 제공한다.**
다른 custom tool 은 JSON schema 와 구현체를 모두 다 직접 만들어야하지만, 기본 제공 Tool 은 JSON Schema(interface) 가 모두 입력되어 있으므로, Claude 가 빠르게 이해할 수 있다.

그리고, Claude model 별로 활용하는 JSON Schema 가 조금씩 다르기 때문에 아래와 같이 분기문이 필요하다.

```python
def get_text_edit_schema(model):
	if model.startswith("claude-3-7-sonnet"):
		return { 
			"type": "text_editor_20250124", 
			"name": "str_replace_editor",
		} 
	elif model.startswith("claude-3-5-sonnet"):
		return { 
			"type": "text_editor_20241022", 
			"name": "str_replace_editor",
		}
```

> 개인적으로 이럴거면 그냥 커스텀 JSON Schema 를 만들어서 사용하는게 더 나을 수도...
### Text Editor Tool
- 파일 또는 디렉토리 내용 보기
- 파일 내 특정 라인 범위 보기
- 파일 내 텍스트 교체

### Web Search Tool
```python
web_search_schema = {
	"type": "web_search_20250305", 
	"name": "web_search", 
	"max_uses": 5,
	"allowed_domains": ["nih.gov"]
}
```

- `max_uses` 필드는 Claude 가 최대로 수행할 수 있는 검색 횟수를 제한해준다. 과도한 API 호출을 방지해주는 수단
- `allowed_domains` 필드는 특정 domain 으로 검색을 제한한다.
