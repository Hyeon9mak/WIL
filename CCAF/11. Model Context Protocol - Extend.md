앞서 설명했듯, Model Context Protocol(MCP) 는 Tool define 및 execute 책임을 Application Server 에서 MCP Server 로 이관시키는 방식이다. (Application Server 는 MCP Client 만 들고 있으면 된다.)

Claude 에게 "내 모든 저장소에서 열려 있는 풀 리퀘스트가 무엇인가요?" 라고 물어볼 수 있는 채팅 애플리케이션을 가정해보자. Claude 는 GitHub API 에 접근할 수 있는 Tool 이 필요하다.

MCP 가 없다면, 우리는 직접 GitHub Tool 전체를 직접 구현해야한다. GitHub 관련 JSON schema 전체를 구현해야한다는 의미. 우리에게 필요한 것 외 모든 인터페이스 명세를 완료해야하고, 필요하다면 구현까지 마무리 해주어야 한다. GitHub API 가 업데이트 됨에 따라 유지보수도 해주어야 함은 물론.

### MCP Server 의 역할
MCP Server 는 외부 서비스에서 구현한 데이터나 기능에 대한 접근을 제공한다. 애플리케이션은 처음부터 모든 것을 구현하는 대신 이 MCP Server 에 연결됨. 결국 tool, prompt, resource 를 표준화된 방식으로 노출하는 전문화된 인터페이스 역할을 하는 셈.

### MCP Client 의 역할
MCP Client 는 Application 과 MCP Server 간 communication bridge 역할을 수행한다.

Transport Agnostic(독립적인) Communication 을 지원하기 때문에, 설정에 따라서 다양한 Protocol 을 이용해서 통신을 진행할 수 있다. 기본적으로 Standard I/O 를 이용한다. HTTP, WebSockets 등등 모두 이용 가능.

### MCP Message Types

- **ListToolsRequest/ListToolsResult**: 어떤 tool 이 있는지
- **CallToolRequest/CallToolResult**: tool 실행 요청과 결과 응답

1. App Server 를 통해 요청이 전달되고, 이를 MCP Client 를 이용해 MCP Server 에게 Tool 목록을 구한다.
2. 그 목록과 함께 질의문을 Claude 에게 전달한다.
3. Claude 가 질의문과 Tool 을 기반으로 어떤 Tool 을 이용할지 응답을 App Server 에게 다시 보낸다.
4. App Server 는 Claude 의 응답을 기반으로 MCP Client 를 활용해 MCP Server 에게 Tool 실행 요청을 보낸다.
5. MCP Server 는 자연스럽게 GitHub API 를 호출하고, 얻어낸 응답 결과를 MCP Client 에게 반환한다.
6. MCP Client 로 얻어낸 실행 결과를 App Server 가 다시 Claude 에게 보낸다.
7. Claude 가 실행 결과와 사용자 요청을 기반으로 마지막 응답을 생성해낸다.

## MCP 를 활용해 Tool 정의하기

공식 Python SDK 를 활용하면 MCP Server 구축이 훨씬 간단해진다. 복잡한 JSON Schema 수동 등록 대신, Decorator pattern 을 활용해서 타입 안정성까지 얻으며 Tool Schema 정의가 가능하다.

- python 언어를 활용하기 때문에 type hint 를 얻으면서 schema 를 명세할 수 있다.
- 파라미터 검증 또한 기대할 수 있다.

```python
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("DocumentMCP", log_level="ERROR")

docs = {
    "deposition.md": "This deposition covers the testimony of Angela Smith, P.E.",
    "report.pdf": "The report details the state of a 20m condenser tower.",
    "financials.docx": "These financials outline the project's budget and expenditures",
    "outlook.pdf": "This document presents the projected future performance of the system",
    "plan.md": "The plan outlines the steps for the project's implementation.",
    "spec.txt": "These specifications define the technical requirements for the equipment"
}
```

사전에 docs 를 대충 준비해두자. 중요한 것은 Schema decorator 다.

```python
@mcp.tool(
	name="read_doc_contents", 
	description="Read the contents of a document and return it as a string."
)
def read_document( 
	doc_id: str = Field(description="Id of the document to read")
):
	if doc_id not in docs: 
		raise ValueError(f"Doc with id {doc_id} not found")
	
	return docs[doc_id]
	
@mcp.tool(
	name="edit_document",
	description="Edit a document by replacing a string in the documents content with a new string."
)
def edit_document(
	doc_id: str = Field(description="Id of the document that will be edited"), 
	old_str: str = Field(description="The text to replace. Must match exactly, including whitespace."), 
	new_str: str = Field(description="The new text to insert in place of the old text.")
): 
	if doc_id not in docs:
		raise ValueError(f"Doc with id {doc_id} not found")
		
	docs[doc_id] = docs[doc_id].replace(old_str, new_str)
```

decorator 는 Tool 의 이름과 설명을 지정하고, 필수 파라미터를 정의할 수 있다. Pydantic Field 는 Claude 가 각각 파라미터에 무엇을 기대하는지 인지할 수 있도록 설명을 제공한다.

## SDK 를 활용해서 MCP Client 구성하기
MCP 클라이언트는 두 가지 주요 구성 요소로 이루어져 있다.

- **MCP Client**: 세션을 더 쉽게 사용하기 위해 우리가 만드는 사용자 정의 클래스
- **Client Session**: 서버에 대한 실제 연결 (MCP Python SDK의 일부)

결국 Client session 이 **connection 을 물고 있는 객체기 때문에, resource 처럼 취급해서 관리**를 해주어야한다. MCP Client 를 통해 이 부분을 자동화 하는 것.

### 핵심 메서드 구현

```python
async def list_tools(self) -> list[types.Tool]:
	result = await self.session().list_tools() 
	return result.tools
```

사용 가능한 모든 Tool 목록 가져오기. 세션에 접근하고 반환한다.

```python
async def call_tool(
	self, tool_name: str, tool_input: dict
) -> types.CallToolResult | None:
	return await self.session().call_tool(tool_name, tool_input)
```

Tool 에 대한 실행을 요청. 결과를 반환한다.

테스트는 다음과 같이 진행하면 된다.

```python
async with MCPClient(
	command="uv", args=["run", "mcp_server.py"]
) as client:
	result = await client.list_tools()
	print(result)
```


## Resource 를 명세해서 Client 에게 편의 제공하기
MCP Server 내의 Resource 를 명세해서 Client 가 가져갈 수 있도록 제공해줄 수 있다. Client 는 이를 가져가서 prompt 에 포함시키고 더 나은 질의문을 전달하는 방식이다.

Client 는 `@document_name` 어노테이션을 통해 원하는 문서에 접근할 수 있다.
- 사용 가능한 모든 문서의 목록 가져오기 (자동완성용)
- 특정 문서를 읽어 내용 가져오기

<img width="1280" height="720" alt="Image" src="https://github.com/user-attachments/assets/f04d04d9-96b1-4cc6-b891-3cdd57a4bde9" />


### MCP Server 에서 명세하는 Resource 종류

- **Direct Resources:** `docs://documents` 와 같이 변하지 않는 정적 URI
- **Templated Resources:** `docs://documents/{doc_id}` 와 같이 매개변수가 있는 URI

<img width="1280" height="720" alt="Image" src="https://github.com/user-attachments/assets/10bcec4e-66d1-4003-8616-829f364caea9" />

### Direct Resource (List Documents)  

```python
@mcp.resource(
    "docs://documents",
    mime_type="application/json"
)
def list_docs() -> list[str]:
    return list(docs.keys())
```

### Templated Resource (Fetch Document)  

```python
@mcp.resource(
    "docs://documents/{doc_id}",
    mime_type="text/plain"
)
def fetch_doc(doc_id: str) -> str:
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]
```

resource 는 문자열, JSON, Binary 등 모든 유형이 될 수 있다. mime_type 매개변수는 Client 에게 반환하는 데이터의 타입을 알려준다.

- `application/json`  - 구조화된 JSON 데이터
- `text/plain` - 순수 텍스트 콘텐츠
- 그 외 다양한 MIME 타입들...


## Resource 를 읽어와서 Claude 에게 알려주기

MCP 클라이언트의 핵심 기능을 위해서는 read_resource 함수가 필요하다.
URI 를 통해서 Resource 를 식별하는 것이 핵심이다.

```python
import json
from pydantic import AnyUrl

async def read_resource(self, uri: str) -> Any:
    result = await self.session().read_resource(AnyUrl(uri))
    resource = result.contents[0]
```

resource 는 다양한 유형의 콘텐츠를 반환할 수 있으므로 Client 가 이를 적절히 발라내야한다.
MIME 유형으로 구별되기 때문에 조건문을 잘 나누면 된다.

```python
if isinstance(resource, types.TextResourceContents):
    if resource.mimeType == "application/json":
        return json.loads(resource.text)
    
    return resource.text
```

## MCP Server 에 Prompt 명세해두기
MCP Server 구성자는 각 Resource 나 Tool 에 대해 잘 알고 있지만, MCP Client 는 그렇지 못할 가능성이 높다. 이에 따라 MCP Server 가 미리 Prompt 를 준비해두면 Client 가 더 높은 수준의 답변을 확보할 가능성이 높다.

<img width="1280" height="720" alt="Image" src="https://github.com/user-attachments/assets/c7d4c1ce-8bb9-4329-89eb-4849591eb19d" />

먼저 기본 message 유형을 가져오고, 함수를 정의한다.

```python
from mcp.server.fastmcp import base

@mcp.prompt(
    name="format",
    description="Rewrites the contents of the document in Markdown format."
)
def format_document(
    doc_id: str = Field(description="Id of the document to format")
) -> list[base.Message]:
    prompt = f"""
Your goal is to reformat a document to be written with markdown syntax.

The id of the document you need to reformat is:

{doc_id}


Add in headers, bullet points, tables, etc as necessary. Feel free to add in extra formatting.
Use the 'edit_document' tool to edit the document. After the document has been reformatted...
"""
    
    return [
        base.UserMessage(prompt)
    ]
```

## MCP Client 에서 Prompt 가져와 활용하기

명세된 Prompt 들은 MCP Client 에서 가져와서 활용할 수 있다.

```python
async def list_prompts(self) -> list[types.Prompt]:
    result = await self.session().list_prompts()
    return result.prompts 

async def get_prompt(self, prompt_name, args: dict[str, str]):
    result = await self.session().get_prompt(prompt_name, args)
    return result.messages
```

prompt 를 조회한 후, 해당 prompt 에서 무엇을 원하는지 (파라미터)를 이해할 필요가 있다.
예를 들어, 문서 포맷팅 prompt 는 `doc_id` 매개변수를 예상할 수 있다.

```python
def format_document(doc_id: str):
    # The doc_id gets interpolated into the prompt
```

Client 에서 `get_prompt` 를 호출할 때, `arguments` 딕셔너리는 예상되는 키를 포함해야 한다.
MCP Server 는 이를 prompt 함수에 키워드 인수로 전달하여 prompt 템플릿에 동적 콘텐츠를 삽입할 수 있게 한다.
