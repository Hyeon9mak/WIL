## Overview of Claude models
크게 3가지. Opus, Sonnet, Haiku
이들의 실질적인 차이점은 최적화 방식에 있다.
지능 / 속도 / 지능 + 속도 밸런스
### Opus
- 가장 높은 수준의 지능
- 여러 단계, 다양한 요구사항, 크고 복잡한 프로젝트를 독립적으로 수행
- 'reasoning' 이라고 부르는 기능을 지원함.
	- 간단한 작업은 빠르게, 어렵고 복잡한 작업은 시간을 투자해서 응답을 준비함.
- 단점은 당연히 시간과 비용.
### Sonnet
- 대부분의 실용적인 사례에 적합한 밸런스.
- 강력한 코딩 능력과 빠른 텍스트 생성 기능.
- 이미 완성된 프로젝트를 개선하는데 가장 적합한 model 로 생각하면 되겠다.
- Opus 와 마찬가지로 'reasoning' 이라고 부르는 기능을 지원함.
### Haiku
- 가장 빠른 model. 응답이 중요한 app 을 위해 특별히 제작됨
- Opus, Sonnet 과 같은 reasoning 능력을 제공하지 않음.
- 대신 속도와 비용에 최적화 되어 있음.
- 실시간 상호 작용이 중요한 app 에 적합하겠다.

결국 중요한 것은 하나의 model 만 고수하는게 아니라, 계속 model 을 바꿔가면서 로직별로 적합한 model 을 차용할 수 있어야 한다. 사용자 응답이 기민한 로직에는 Haiku 를, 깊은 reasoning 이 복잡한 비즈니스 로직에는 Opus 를 채택하는 등...
물론 대부분의 경우에 Sonnet 이 가장 보편적인 선택이 되겠다.
