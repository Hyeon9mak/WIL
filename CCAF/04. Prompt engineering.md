평가 파이프라인의 평가기에 `extra_criteria` 를 추가하면 구체적인 기준(채점 기준)을 추가해줄 수 있다.
"출력에 일일 총 칼로리, 거시 영양소 분해, 정확한 음식, 양, 타이밍 등을 포함해라" 식의 추가가 가능하다.

prompt = f"""

## 명확하고 직접적이게.
- 2가지 규칙은 prompt 의 첫 번째 줄에 가장 강력히 적용 되어야 한다.
- **prompt 의 첫 문장이 사실 모든 걸 결정한다 해도 과언이 아니다.**
	- 첫 문장에서 가장 간결하고 직접적인 언어를 사용해야 한다.
	- 행동 동사를 통해 Claude 가 무얼 해야하는지 정확히 알려줄 것.
	- "태양광 패널이 어떻게 동작하는지 표로 작성해주세요."

**어떤 행동을 취할지, 무얼 만들지, 주요 제약 조건**

prompt 를 다음과 같이 개선해보자.

```python
prompt = f"""
	Generate a one-day meal paln for an athlete that meets their dietary restrictions.
	
	- Height: {prompt_inputs["height"]}
    - Weight: {prompt_inputs["weight"]}
	- Goal: {prompt_inputs["goal"]}
	- Dietary restrictions: {prompt_inputs["restrictions"]}
"""
```

평가기를 돌리면 3.92점짜리 prompt 로 평가된다.

## 구체적으로 말하기.
- 모든 것을 model 의 해석에 맡기지 않고, Claude 가 원하는 결과물로 안내할 수 있도록 명확한 가이드라인이나 단계를 제공하는 것.
- 보통 2가지 패턴의 가이드라인을 많이 제공하게 될 것이다.
	- 출력(output)에 포함시켜야하는 내용 지시
	- 최종 결과물에 어떻게 도달해야하는지 작업 순서 지시
	- 물론 2가지를 합친 prompt 도 당연히 유효하다.

<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/9ca86c22-aa07-4453-95d5-592e7911ef86" />

prompt 를 다음과 같이 개선해보자.

```python
prompt = f"""
	Generate a one-day meal plan for an athlete that meets their dietary restrictions.
	
	- Height: {prompt_inputs["height"]}
    - Weight: {prompt_inputs["weight"]}
	- Goal: {prompt_inputs["goal"]}
	- Dietary restrictions: {prompt_inputs["restrictions"]}
	  
	Guidelines:
	1. Include accurate daily calorie amount
	2. Show protein, fat, and carb amounts
	3. Specify when to eat each meal
	4. Use only foods that fit restrictions
	5. List all portion sizes in grams
	6. Keep budget-friendly if mentioned
"""
```

7.3 점 평가를 받게 된다. 엄청나게 큰 개선 효과.
Claude 가 거쳐야 할 순서를 나열해준 덕분에 정확도가 높아진 것.

- 출력 가이드라인은 항상 사용하는 것이 좋다.
- 단계 가이드라인은 복잡한 문제를 해결하는데 가치가 높다.

## XML tag 를 활용한 구조

우리가 prompt 에 예시, 콘텐츠 등을 삽입할 때. 적은 양을 삽입할 땐 별다른 문제가 없다.
하지만 작업 규모가 크고 복잡해질수록 Claude 가 문자열 뭉텅이를 보고 무엇에 가치를 두어야하는지 판단하기 어렵고 context 가 쉽게 더럽혀진다.

어떤 부분에 집중해야하는지를 XML tag 를 통해 마킹해주는 것.
**꼭 공식 XML tag 를 이용할 필요가 없다! 콘텐츠에 어울리는 형태를 이용하면 된다.**
**오히려 tag 이름이 구체적일수록 Claude 가 목적을 더 잘 이해할 수 있다.**

<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/5f1f0578-d30c-4852-b6fd-81418210ca03" />

prompt 를 다음과 같이 개선해보자.

```python
prompt = f"""
	Generate a one-day meal plan for an athlete that meets their dietary restrictions.
	
	<athlete_information>
	- Height: {prompt_inputs["height"]}
    - Weight: {prompt_inputs["weight"]}
	- Goal: {prompt_inputs["goal"]}
	- Dietary restrictions: {prompt_inputs["restrictions"]}
    </athlete_information>
	  
	Guidelines:
	1. Include accurate daily calorie amount
	2. Show protein, fat, and carb amounts
	3. Specify when to eat each meal
	4. Use only foods that fit restrictions
	5. List all portion sizes in grams
	6. Keep budget-friendly if mentioned
"""
```

7.94 점의 평가를 얻을 수 있다.

## 예시 제공
무엇보다 가장 강력한 방식. 예시를 알려주면 Claude 는 그 무엇보다 좋은 결과물을 만들어 낼 수 있다.
"one-shot prompting", "multi-shot prompting" 이라고도 불린다.

one-shot prompting 의 경우 당연히 단일 근거기 때문에 Claude 가 오류를 범할 여지가 남아있다.
예시가 많은 multi-shot prompting 을 준비하면 정확도가 자연스럽게 올라간다.

<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/c58f1536-f79c-4768-b496-8f86f27ac81b" />

- 엣지 케이스, 복잡한 출력 형식, 모호한 입력을 다루는 경우에 가장 효과적인 방식.
- test_case 들 중 가장 높은 prompt 점수를 획득한 결과물을 예시로 삼는 것 또한 꿀팁 중 하나다.
- 단순히 예시만 주는게 아니라, 예시에 맥락을 추가해서 설명해주면 좋은 답변의 이유를 이해하는 데에도 도움이 된다.
